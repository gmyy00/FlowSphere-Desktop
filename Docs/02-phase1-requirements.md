# Phase 1 需求文档：本地待办日程 Demo

## 1. 基本信息

| 项目 | 说明 |
|---|---|
| 产品名称 | FlowSphere Desktop Demo |
| 阶段 | Phase 1 |
| 类型 | 本地桌面应用 |
| 目标平台 | Windows 优先 |
| 技术方向 | Python + PySide6 |
| 数据存储 | 本地 JSON 文件 |
| 网络依赖 | 无 |
| 账号系统 | 无 |

## 2. 功能范围

### 2.1 模块清单

| 模块 | 优先级 | 说明 |

## 3. 页面与交互

### 3.1 悬浮卡片

Demo 阶段主界面优先保证可运行和可验收，可以使用普通桌面窗口。窗口内部采用双标签页：

```text
FlowSphere Desktop
├── 待办
│   ├── 新增按钮
│   ├── 待办列表
│   └── 编辑/删除/完成操作
└── 日程
    ├── 新增按钮
    ├── 日程列表
    └── 编辑/删除操作
```

窗口行为：

| 行为 | 说明 |
|---|---|
| 普通窗口 | Demo 可使用系统默认标题栏，降低 UI 调试成本 |
| 固定尺寸 | 先使用固定尺寸，后续再做尺寸档位 |
| 双标签页 | 待办和日程分开管理 |
| 列表操作 | 列表内提供新增、编辑、删除、完成等操作 |

系统托盘行为不纳入 Demo 必须验收。如时间允许再实现：

| 行为 | 说明 |
|---|---|
| 点击卡片关闭按钮 | 隐藏悬浮卡片，不退出程序 |
| 点击系统关闭按钮 | 隐藏悬浮卡片，不退出程序 |
| 右键托盘图标 | 显示菜单 |
| 菜单：显示窗口 | 重新显示并聚焦悬浮卡片 |
| 菜单：关闭 | 完全退出应用 |
| 左键托盘图标 | 显示/隐藏悬浮卡片 |

### 3.2 待办列表展示字段

| 字段 | 说明 |
|---|---|
| 标题 | 待办事项标题 |
| 截止时间 | deadline 格式化文本 |
| 提醒时间 | remind_at 格式化文本 |
| 状态 | 未完成 / 已完成 |
| 操作 | 编辑、删除、完成/取消完成 |

### 3.3 日程列表展示字段

| 字段 | 说明 |
|---|---|
| 标题 | 日程标题 |
| 开始时间 | start_time 格式化文本 |
| 结束时间 | end_time 格式化文本，可为空 |
| 提醒时间 | remind_at 格式化文本 |
| 状态 | 未开始 / 进行中 / 已结束 |
| 操作 | 编辑、删除 |

### 3.4 新增与编辑弹窗

待办弹窗字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| title | 是 | 待办标题 |
| description | 否 | 备注 |
| deadline | 否 | 截止时间 |
| remind_at | 否 | 提醒时间 |

日程弹窗字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| title | 是 | 日程标题 |
| description | 否 | 备注 |
| start_time | 是 | 开始时间 |
| end_time | 否 | 结束时间 |
| remind_at | 否 | 提醒时间 |

## 4. 业务规则

### 4.1 待办规则

| 编号 | 规则 |
|---|---|
| TODO-01 | 待办标题不能为空 |
| TODO-02 | 新建待办默认 `done = false` |
| TODO-03 | 已完成待办不再触发提醒 |
| TODO-04 | 删除待办需要二次确认 |
| TODO-05 | 截止时间早于当前时间时允许保存，但 UI 标记为逾期 |
| TODO-06 | 提醒时间为空时不提醒 |

### 4.2 日程规则

| 编号 | 规则 |
|---|---|
| SCH-01 | 日程标题不能为空 |
| SCH-02 | 日程开始时间不能为空 |
| SCH-03 | 如果填写结束时间，结束时间不能早于开始时间 |
| SCH-04 | 已结束日程保留在列表中，但显示为已结束 |
| SCH-05 | 提醒时间为空时不提醒 |
| SCH-06 | 一期不做日程冲突检测 |

### 4.3 提醒规则

| 编号 | 规则 |
|---|---|
| REM-01 | 应用运行期间每 30 秒检查一次提醒 |
| REM-02 | 当前时间大于等于 `remind_at` 且未提醒过时触发提醒 |
| REM-03 | 每条记录需要保存 `reminded` 标记，避免重复提醒 |
| REM-04 | 待办完成后不再提醒 |
| REM-05 | 一期只做单次提醒，不做重复提醒 |

## 5. 数据模型

### 5.1 Todo

```json
{
  "id": "uuid",
  "title": "完成 Demo",
  "description": "实现本地待办和日程",
  "deadline": "2026-06-10T18:00:00",
  "remind_at": "2026-06-10T17:30:00",
  "done": false,
  "reminded": false,
  "created_at": "2026-06-09T10:00:00",
  "updated_at": "2026-06-09T10:00:00"
}
```

### 5.2 Schedule

{

## 6. 本地文件

默认数据目录：

```text
data/
├── todos.json
└── schedules.json
```

文件格式：

```json
[
  {
    "id": "uuid",
    "title": "example"
  }
]
```

## 7. 异常处理

| 场景 | 处理方式 |
|---|---|
| 数据文件不存在 | 自动创建空数组文件 |
| JSON 格式损坏 | 提示用户并备份损坏文件 |
| 保存失败 | 弹窗提示并写日志 |
| 提醒触发失败 | 写日志，不阻塞主程序 |
| 时间字段非法 | 表单校验并阻止保存 |

## 8. 一期验收效果

演示时应能完成：

1. 启动 FlowSphere Desktop Demo。
2. 新增一个待办，设置截止时间和提醒时间。
3. 新增一个日程，设置开始时间和提醒时间。
4. 编辑待办标题。
5. 标记待办完成。
6. 删除一个日程。
7. 等待提醒时间到达，看到提醒。
8. 关闭并重启应用，确认数据仍在。
