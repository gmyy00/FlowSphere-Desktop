# 测试计划与验收标准

## 1. 测试目标

一期测试目标是保证本地 Demo 的核心闭环稳定：

| 目标 | 说明 |
|---|---|
| 数据不丢 | 新增、编辑、删除后 JSON 正确保存 |
| 操作可用 | 待办和日程的基础操作没有阻断问题 |
| 提醒可靠 | 到期提醒能触发且不重复触发 |
| 异常可控 | 文件不存在、JSON 损坏等场景有处理 |
| 演示顺畅 | 从启动到重启恢复数据的流程可完整演示 |

## 2. 测试分层

| 层级 | 工具 | 范围 |
|---|---|---|
| 单元测试 | pytest | 模型、服务、存储、提醒判断 |
| 集成测试 | pytest + 临时目录 | 服务组合和 JSON 文件读写 |
| 手工测试 | Windows 桌面 | UI 操作、提醒弹窗、演示流程 |

## 3. 单元测试用例

### 3.1 StorageService

| 编号 | 用例 | 预期 |
|---|---|---|
| ST-01 | 数据文件不存在时读取 todos | 返回空数组并创建文件 |
| ST-02 | 保存 todos 后再次读取 | 数据一致 |
| ST-03 | 保存 schedules 后再次读取 | 数据一致 |
| ST-04 | JSON 文件为空 | 返回空数组或提示可恢复错误 |
| ST-05 | JSON 格式损坏 | 备份损坏文件并返回空数组或错误结果 |
| ST-06 | 写入失败 | 抛出可识别异常并记录日志 |

### 3.2 TodoService

| 编号 | 用例 | 预期 |
|---|---|---|
| TD-01 | 创建标题合法的待办 | 创建成功，`done=false` |
| TD-02 | 创建标题为空的待办 | 创建失败 |
| TD-03 | 编辑待办标题 | 标题更新，`updated_at` 更新 |
| TD-04 | 删除存在的待办 | 删除成功 |
| TD-05 | 删除不存在的待办 | 返回失败或无副作用 |
| TD-06 | 标记待办完成 | `done=true` |
| TD-07 | 取消完成待办 | `done=false` |
| TD-08 | 已完成待办查询待提醒列表 | 不返回该待办 |

### 3.3 ScheduleService

| 编号 | 用例 | 预期 |
|---|---|---|
| SC-01 | 创建合法日程 | 创建成功 |
| SC-02 | 创建标题为空的日程 | 创建失败 |
| SC-03 | 创建无开始时间的日程 | 创建失败 |
| SC-04 | 结束时间早于开始时间 | 创建失败 |
| SC-05 | 编辑日程时间 | 更新时间正确 |
| SC-06 | 删除存在日程 | 删除成功 |
| SC-07 | 判断未开始日程状态 | 返回 `upcoming` |
| SC-08 | 判断进行中日程状态 | 返回 `active` |
| SC-09 | 判断已结束日程状态 | 返回 `ended` |

### 3.4 ReminderService

| 编号 | 用例 | 预期 |
|---|---|---|
| RM-01 | 当前时间未到提醒时间 | 不触发 |
| RM-02 | 当前时间等于提醒时间 | 触发 |
| RM-03 | 当前时间晚于提醒时间且未提醒 | 触发 |
| RM-04 | `reminded=true` | 不触发 |
| RM-05 | 待办 `done=true` | 不触发 |
| RM-06 | 提醒触发后保存 `reminded=true` | 下次不重复提醒 |

## 4. 集成测试用例

| 编号 | 流程 | 预期 |
|---|---|---|
| IT-01 | 创建待办 -> 保存 -> 新服务实例读取 | 数据存在 |
| IT-02 | 创建日程 -> 保存 -> 新服务实例读取 | 数据存在 |
| IT-03 | 创建提醒时间已到的待办 -> 扫描提醒 | 返回待提醒项并更新状态 |

## 7. 演示验收脚本

演示时按以下步骤执行：

1. 启动应用。
2. 新增待办 `完成 FlowSphere Demo`，设置 1 分钟后提醒。
3. 新增日程 `项目复盘会议`，设置 2 分钟后提醒。
4. 编辑待办描述。
5. 标记另一个待办为完成。
6. 等待第一个提醒弹出。
7. 删除一个日程。
8. 关闭应用。
9. 重新启动应用。
10. 确认数据仍然存在，且已提醒记录不重复提醒。
