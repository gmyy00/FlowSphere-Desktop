# 开发周期与里程碑

## 1. 一期周期建议

一期本地 Demo 建议按 10 个开发日规划。如果每天投入时间较少，可以拉长到 2 到 3 周。

| 阶段 | 时间 | 目标 |
|---|---|---|
| D1 | 0.5-1 天 | 初始化项目结构和运行环境 |
| D2-D3 | 2 天 | 完成数据模型和本地存储 |
| D4-D5 | 2 天 | 完成待办管理 |
| D6-D7 | 2 天 | 完成日程管理 |
| D8 | 1 天 | 完成提醒扫描和提醒弹窗 |
| D9 | 1 天 | 完成异常处理、日志、基础 UI 优化 |
| D10 | 1 天 | 测试、修复、整理 README 和演示流程 |

## 2. 任务拆分

### 2.1 初始化

| 任务 | 输出 |
|---|---|
| 创建 `desktop/` 目录 | 项目骨架 |
| 创建虚拟环境 | Python 环境 |
| 添加 `requirements.txt` | 依赖清单 |
| 创建 `main.py` | 可启动空窗口 |

验收：运行命令后出现空白悬浮卡片窗口。

### 2.2 数据层

| 任务 | 输出 |
|---|---|
| 实现 Todo dataclass | 待办模型 |
| 实现 Schedule dataclass | 日程模型 |
| 实现 StorageService | JSON 读写 |
| 处理文件不存在 | 自动创建文件 |
| 处理 JSON 损坏 | 备份损坏文件 |

验收：单元测试能创建、读取、保存 todos 和 schedules。

### 2.3 待办模块

| 任务 | 输出 |
|---|---|
| 实现 TodoService | 待办 CRUD |
| 实现 TodoTab | 待办列表 |
| 实现 TodoDialog | 新增编辑弹窗 |
| 实现完成/取消完成 | 状态切换 |
| 实现删除确认 | 防误删 |

验收：用户可完整新增、编辑、删除、完成待办。

### 2.4 日程模块

| 任务 | 输出 |
|---|---|
| 实现 ScheduleService | 日程 CRUD |
| 实现 ScheduleTab | 日程列表 |
| 实现 ScheduleDialog | 新增编辑弹窗 |
| 实现时间校验 | 结束时间不能早于开始时间 |
| 实现状态展示 | 未开始、进行中、已结束 |

验收：用户可完整新增、编辑、删除日程。

### 2.5 提醒模块

| 任务 | 输出 |
|---|---|
| 实现 ReminderService | 提醒扫描 |
| 接入 QTimer | 周期执行 |
| 实现提醒弹窗 | 到期提示 |
| 保存 reminded 标记 | 避免重复提醒 |

验收：设置 1 分钟后的提醒，到点后出现提醒，且只提醒一次。

### 2.6 收尾

| 任务 | 输出 |
|---|---|
| 补充 pytest 测试 | 核心逻辑可验证 |
| 优化 UI 文案 | 演示更清楚 |
| 添加 README | 运行说明 |
| 走完整演示流程 | 验收通过 |

## 4. 优先级控制

必须完成：

| 功能 | 原因 |
|---|---|
| 待办 CRUD | 核心功能 |
| 日程 CRUD | 核心功能 |
| 本地持久化 | Demo 可信度 |
| 提醒能力 | 产品差异点 |

可以延期：

| 功能 | 原因 |
|---|---|
| 系统托盘 | 不影响核心闭环 |
| AI 输入 | 有额外模型依赖 |
| 主题切换 | UI 增强，不是核心逻辑 |
| 打包 exe | 源码运行即可先验收 |

## 5. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| PySide6 UI 调试耗时 | 进度延迟 | 先实现无边框悬浮卡片和基础控件，复杂动画后续再做 |
| 时间选择控件细节多 | 表单复杂 | 使用 QDateTimeEdit，避免自定义控件 |
| 系统通知兼容性 | 提醒失败 | 一期使用应用内 QMessageBox 兜底 |
| JSON 文件损坏 | 数据丢失 | 写入前备份或临时文件替换 |
| 功能范围膨胀 | Demo 延迟 | AI、设置页、主题、开机自启全部放后续，托盘不阻塞验收 |

## 6. 一期完成后的下一步

一期完成后，根据剩余时间选择：

1. 如果时间少：补测试、补 README、录制演示流程。
2. 如果时间中等：增加系统托盘。
3. 如果时间充足：整理 UI 细节和打包准备，不在 Demo 阶段接入 Ollama。
